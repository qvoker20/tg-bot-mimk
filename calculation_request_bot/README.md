# Calculation Request Bot (MIM-K)

Окремий Telegram-бот для короткого сценарію:
1. Ім'я для звертання
2. Проєкт (текст/файл/фото/PDF/архів, можна кілька файлів)
3. Контактний номер телефону

Після цього звернення зберігається в PostgreSQL і (опційно) дублюється менеджеру в Telegram.

## Що зберігається

По кожному зверненню:
- дата і час старту
- Telegram ID
- Telegram username (якщо є)
- ім'я, яке ввів користувач
- перше повідомлення або файл (JSON payload)
- усі надіслані матеріали (JSON payload array)
- локальні шляхи до збережених файлів
- дата і час надсилання проєкту
- контактний номер телефону
- шлях до ZIP-архіву матеріалів

## Куди зберігаються файли

Файли з Telegram завантажуються локально у папку:
- calculation_request_bot/aload/request_<ID_заявки>/

Після завершення заявки бот збирає ZIP-архів у:
- calculation_request_bot/aload/archives/request_<ID_заявки>.zip

Також у папці заявки створюється payloads.json з переліком усіх надісланих матеріалів.

## Змінні .env

Потрібні:
- PG_HOST
- PG_PORT
- PG_DBNAME
- PG_USER
- PG_PASSWORD
- CALCULATION_BOT_TOKEN

Опційно:
- CALC_MANAGER_CHAT_ID (chat_id менеджера для сповіщення про завершений запит)

## Запуск

З кореня репозиторію:

```bash
pip install -r calculation_request_bot/requirements.txt
python calculation_request_bot/bot.py
```

## SQL схема

Схема таблиці в:
- sql/project_calculation_requests.sql

Бот також сам виконує автоперевірку/створення таблиці при старті.

## Логіка, якщо клієнт спочатку надсилає файл

Сценарій підтримується:
- бот фіксує перший файл/повідомлення
- все одно просить ім'я
- після імені дає змогу дозавантажити ще файли і натиснути "Завантажити"
- потім просить номер телефону
